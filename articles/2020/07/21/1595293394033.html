1、使用命令行连接到MYSQL数据库

首先，你需要查看连接到具体哪一个服务器，如果连接的不是本机服务器的话，那就需要安装mysql-client 客户端应用程序才可以进行连接，连接口令如下：

shell> mysql -h localhost -P 3306 -u <username> -p<password>
shell> mysql --host=localhost --port=3306 --user=root --password=<password>
shell> mysql --host localhost --port 3306 --user root --password=<password>

1.-P（大写）用于指定端口号

2.-p（小写）用于指定密码

3.-p（小写）参数后没有空格。

4.--password=后面没有空格，连接时要格外注意

5.-A参数，如果不使用的话，在使用 use dbname 语句时，会读取数据库中所有表的信息和列的完整名称，所以如果读取的是一个数据很多的数据库时，-A个人建议是一定要加上，不加的话，很有可能会卡死服务器。mysql在我们不使用这个登陆的时候，也会给与我们提示

Reading table information for completion of table and column names
You can turn off this feature to get a quicker startup with -A

默认连接情况下，host为 localhost，端口号为 3306

在连接mysql时，强烈建议不要用明文密码进行连接MYSQL数据库，其他替代方法在之后的博客中进行讨论。使用 mysql -u <username> -p进行连接是最为常用的方式。

[root@zabbix /usr/local/nginx-1.18/conf]# mysql -u root -p123456
mysql: [Warning] Using a password on the command line interface can be insecure.

在进行查询时，可以以 ；|\G|\g|进行结尾，一般的sql语句以**;** 结尾，\g输出以水平显示，\G输出以垂直显示。

mysql> select user,host from mysql.user where user='root'\G;
*************************** 1. row ***************************
user: root
host: %
*************************** 2. row ***************************
user: root
host: localhost
2 rows in set (0.00 sec)

mysql> select user,host from mysql.user where user='root'\g;
+------+-----------+
| user | host      |
+------+-----------+
| root | %         |
| root | localhost |
+------+-----------+
2 rows in set (0.00 sec)

2、创建数据库,表

数据库定义语言（DDL：）

其语句包括动词CREATE和DROP。在数据库中创建新表或删除表（CREAT TABLE 或 DROP TABLE）为表加入索引等。简而言之，对数据进行维护和管理的语句，就是DDL

数据库是许多表的集合，而数据库服务器可以容纳许多这样的数据库。逻辑关系如下：

数据库服务器->数据库->表（由列定义）->行

数据库和表称为数据库对象。任何操作(如创建、修改或删除数据库对象)都称为数据定义语言(DDL:Data Definition Language)操作.

数据按某种蓝图组织构建数据库（分为数据库和表），这种数据的组织形式被称为 schema。

2.2 创建表

mysql> CREATE TABLE IF NOT EXISTS `company`.`customers` (
`id` int unsigned AUTO_INCREMENT PRIMARY KEY,
`first_name` varchar(20),
`last_name` varchar(20),
`country` varchar(20)
) ENGINE=InnoDB;
Query OK, 0 rows affected (0.20 sec)
  • 句点符号：表可以使用database.table引用。如果巳经连接到数据库，则可以简单地使用customers而不是company.customers
  • IF NOT EXISTS: 如果存在一个具有相同名字的表，并且你指定了这个子句， MySQL只会抛出一个警告，告知表已经存在。否则，MySQL将抛出一个错误。
  • id : 它被声明为一个整型数，因为它只包含整型数。除此之外，还有两个关键字，AUTO INCREMENT和PRIMARY KEY。(自动增长和主键约束)
  • AUTO_INCREMENT: 自动生成线性递增序列， 因此不必担心为每一行的id分配值。
  • PRIMARY KEY: 主键约束：每行都由一个非空的UNIQUE列标识。 只有一列应该在表中定义。 如果一个表包含AUTO_INCREMENT列，则它会被视为PRIMARY KEY。first_name、 last_name和country: 它们包含字符串， 因此它们被定义为 varchar类型。
  • Engine: 存储引擎。一些类型的存储引擎包括InnoDB、 MyISAM、FEDERATED、BLACKHOLE、CSV和MEMORY。在所有引擎中，InnoDB 是唯一的事务引擎，也是默认引擎。

3、插入，删除，修改

INSERT、UPDATE、DELETE和SELECT操作称为 数据操作语言（DML：Data Manipulation Language）INSERT、 UPDATE和DELETE也称为写操作，或者简称为写(write)。SELECT是一个读操作，简称为读(read)。

3.1 插入

向company数据库中的 customers表 插入 first_name, last_name,country 三个字段的数据

mysql> INSERT IGNORE INTO `company`.`customers`(first_name, last_name,country)
VALUES 
('Mike', 'Christensen', 'USA'),
('Andy', 'Hollands', 'Australia'),
('Ravi', 'Vedantam', 'India'),
('Rajiv', 'Perera', 'Sri Lanka');

Query OK, 4 rows affected (0.00 sec)
Records: 4  Duplicates: 0  Warnings: 0

插入4个字段，包含主键

mysql> INSERT IGNORE INTO `company`.`customers`(id, first_name, last_name,country)
VALUES 
(1, 'Mike', 'Christensen', 'USA'),
(2, 'Andy', 'Hollands', 'Australia'),
(3, 'Ravi', 'Vedantam', 'India'),
(4, 'Rajiv', 'Perera', 'Sri Lanka');

Query OK, 4 rows affected (0.00 sec)
Records: 4  Duplicates: 0  Warnings: 0

INSERT IGNORE:如果该行已有数据，则新插入的数据将被忽略，INSERT语句会执行成功，但是会生成警告信息，同时生成一个重复数据的数目，如果没有输入IGNORE语句的话，INSERT语句会插入失败，每一行的唯一性由主键标识。

有就忽略，没有就插入

mysql> INSERT IGNORE INTO `company`.`customers`(id, first_name, last_name,country) VALUES  (1, 'Mike', 'Christensen', 'USA'), (2, 'Andy', 'Hollands', 'Australia'), (3, 'Ravi', 'Vedantam', 'India'), (4, 'Rajiv', 'Perera', 'Sri Lanka');
Query OK, 0 rows affected, 4 warnings (0.00 sec)

INSERT INTO:插入数据，数据库会检查主键（Primary Key），如果出现重复会报错；简单来说：

有则插入，无则报错

mysql> INSERT INTO `company`.`customers`(id, first_name, last_name,country)
VALUES 
(1, 'Mike', 'Christensen', 'USA'),
(2, 'Andy', 'Hollands', 'Australia'),
(3, 'Ravi', 'Vedantam', 'India'),
(4, 'Rajiv', 'Perera', 'Sri Lanka');
ERROR 1062 (23000): Duplicate entry '1' for key 'customers.PRIMARY'

REPLACE

该关键字的作用是当插入的记录遇到主键或者唯一键重复时先删除表中重复的记录行再插入，即

有则删除+插入，无则插入

语法为：

mysql> REPLACE INTO customers VALUES (1,'Mike','Christensen','America');
Query OK, 2 rows affected (0.00 sec)

3.2 更新

更新行记录

不加where的话，update更新字段会更新整个表！！！切记必须要加上where进行操作

mysql> UPDATE customers SET first_name='Rajiv', country='UK' WHERE id=4;
Query OK, 1 row affected (0.01 sec)
Rows matched: 1  Changed: 1  Warnings: 0

mysql> select * from customers;
