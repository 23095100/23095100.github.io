2、使用二进制进行安装

  YUM/APT 进行安装可能会出现与其他已安装软件冲突的情况，并且会要求安装其他一些依赖项，这个时候我们可以选择通过官方给出的二进制文件进行安装

  这里我们以 社区版 （MySQL Community Server）mysql-5.7.22-linux-glibc2.12-x86_64 进行示例安装

  其他版本可以去官方网站下载 https://dev.mysql.com/downloads/mysql/

2.1 基于 YUM 的系统 二进制安装 MySQL

  2.1.1 安装 libaio 依赖库

1root@DESKTOP-KK8078I:/tmp# sudo yum -y install libaio1

2.2.2 从 MySQL 官方网站下载 MySQL 二进制文件

2.2 基于 APT 的系统 二进制安装 MySQL

  2.2.1 安装 libaio 依赖库

1root@Ubuntu20:/tmp# sudo apt-get install libaio1

  https://dev.mysql.com/downloads/mysql/

  2.2.10 将目录导入到环境变量

$PATH：决定了 shell 将到哪些目录中寻找命令或程序，PATH 的值是一系列目录，当您运行一个程序时，Linux 在这些目录下进行搜寻编译链接。

  修改 $PATH 的方法有很多，比如：

1root@DESKTOP-KK8078I:/usr/local/mysql# export PATH=$PATH:/usr/local/mysql/bin
2root@DESKTOP-KK8078I:/usr/local/mysql# echo $PATH
3/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:/snap/bin:/usr/local/mysql/bin

  但是这种方式修改的环境变量，在终端关闭后就会失效，所以不推荐此方式进行修改。

  第二种：编辑 /etc/profile 文件，进行修改环境变量

1root@DESKTOP-KK8078I:/usr/local/mysql/bin# vim /etc/profile

  添加以下内容，并更新文件

1MYSQL_HOME=/usr/local/mysql
2PATH=$PATH:$MYSQL_HOME/bin
3export PATH MYSQL_HOME
4root@DESKTOP-KK8078I:/usr/local/mysql/bin# source /etc/profile

  就算是更新完成了

3、systemd 集成 MySQL

Systemd 是一个系统管理守护进程、工具和库的集合，用于取代 System V 初始进程。Systemd 的功能是用于集中管理和配置类 UNIX 系统。

Systemctl 是一个 systemd 工具，主要负责控制 systemd 系统和服务管理器。

  如果使用 YUM/APT 安装 MySQL 的话，会自动生成 systemd 目录，而二进制文件安装的话，系统中并没有集成，建议手动添加 systemd 选项。

  3.1 新建文件夹，并编辑文档

1root@DESKTOP-KK8078I:/usr/local/mysql/bin# sudo mkdir -pv /etc/systemd/system/mysqld.service.d
2mkdir: created directory '/etc/systemd/system/mysqld.service.d'
3root@DESKTOP-KK8078I:/usr/local/mysql/bin# sudo vim /etc/systemd/system/mysqld.service.d/override.conf

  3.2 添加以下内容

1[Service]
2LimitNOFILE=max_open_files
3Nice=nice_level
4LimitCore=core_file_limit
5Environment="LD_PRELOAD=/path/to/malloc/library"
6Environment="TZ=time_zone_setting"

  3.3 重新加载 systemd 配置，并启动 MySQL

 1root@DESKTOP-KK8078I:/usr/local/mysql/bin# systemctl daemon-reload
 2root@DESKTOP-KK8078I:/usr/local/mysql/bin# systemctl start mysql

4、MySQL 启动，关闭，重启等操作

  如果您进行上一步的话，那就可以通过 systemctl 进行管理 MySQL 服务的启动，关闭，重启等操作。

4.1 基于 YUM 的系统

 1启动mysql
 2root@DESKTOP-KK8078I:/usr/local/mysql/bin# systemctl start mysqld
 3停止mysql
 4root@DESKTOP-KK8078I:/usr/local/mysql/bin# systemctl stop mysqld
 5重启mysql
 6root@DESKTOP-KK8078I:/usr/local/mysql/bin# systemctl restart mysqld
 7开机启动mysql
 8root@DESKTOP-KK8078I:/usr/local/mysql/bin# systemctl enable mysqld
 9关闭开机启动
10root@DESKTOP-KK8078I:/usr/local/mysql/bin# systemctl disable mysqld

4.2 基于 APT 的系统

 1启动mysql
